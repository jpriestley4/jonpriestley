Project Overview

The brief

The goal of this project was to design a web application that supports amateur investors in interpreting financial statements.

Role

  • UX Designer

Duration

  • 6 Months

Skills

Key Features

  • The page navigation toolbar has a smooth scrolling effect so that users can efficiently skip to the content they want.
  • Filters are used on the ‘Grids’, ‘Space’ and ‘Responsive' pages, allowing users to sift through content.
  • On the ‘Colour’ page, interactive colour wheels provide a visually engaging way of understanding different colour schemes.
  • A dynamic data array is used on the ‘Responsive Design’ page to allow users to browse through common screen resolutions by device type.
  • The ‘Blogs’ page uses an Application Programming Interface (API) to pull relevant articles and blogs.
  • The ‘Icons Quiz’ page provides a fun quiz that lets users test their understanding of iconography.
  • Interactive flip cards are used on the ‘Space’ page.
  • Videos are embedded in the ‘Videos’ and ‘Grids’ pages.
  • Slideshows are used on the ‘Grids’ and ‘Typography’ pages.

Functionality

Global Navigation

The global navigation bar provides links to supplementary resource pages. These links are styled with a Hover CSS property, providing feedback to communicate that these are clickable links. In addition, the site’s logo is placed on the left hand side of the global navigation which acts as a link to the home page. This aligns with standard web design practice as it orientates users around the purpose of the site (Whitenton, 2016). Since the logo is quite large, media queries are used to create a responsive logo which accommodates different screen sizes. This responsive design provides white space to assist with the legibility of the navigation links.

Home page

The body of the Home page includes links to all the design guides that represent the main content on the site. Each guide was styled with a concise header that contrasts against a distinctly styled background. These headers and images give an indication of the content underneath each link, providing useful information scent to the user (Budiu, 2020). When a user hovers over each guide, a ‘box-shadow’ and ‘scaling’ effect appears. This prompts users to click on the guides. One difficulty when creating these links was preventing the background images from tiling when the browser window changes in size. This was easily resolved by using the background-repeat: none CSS property. In addition, media queries are used so that the guides are responsive to different screen sizes.
